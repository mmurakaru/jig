# Write a failing test

Turn the previous handoff's reproduction into a test: it goes red on the
current code for the same reason the reproduction failed, and will go green
once the bug is fixed. Run it to confirm the red. Do not fix the bug.

Done when the suite has exactly one new test, red for the bug's reason -
your summary names it, how to run it, and what it asserts; list it as an
artifact. fail = the bug cannot be expressed as a test.
